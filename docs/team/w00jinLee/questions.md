# 질문 1

 “adder.c에서 왜 그냥 printf()로 바로 출력하지 않고, 굳이 sprintf()로 content 문자열을 먼저 만들어야 하나요? 그리고 content에 =로 문자열을 넣으면 안 되나요?” (CSAPP 3판 920페이지)

# 질문 1 답변
“sprintf()를 꼭 써야 하는 건 아닙니다. printf()로 바로 출력해도 HTTP 응답은 만들 수 있습니다.
다만 그 예제에서는 Content-Length 헤더에 본문 길이를 넣으려고 했기 때문에, 먼저 content에 응답 본문을 전부 만들어 두고 strlen(content)로 길이를 계산한 뒤 출력한 것입니다.

또 content가 char content[MAXLINE];처럼 선언된 문자 배열이라면, 선언이 끝난 뒤에

content = "hello";

처럼 =로 문자열 전체를 다시 대입할 수는 없습니다.
C에서 배열은 통째로 대입하는 대상이 아니기 때문입니다. 그래서 문자열을 넣거나 바꾸려면 strcpy, sprintf, snprintf 같은 함수를 사용해야 합니다.”

정리:

“즉, sprintf()를 쓴 이유는 본문을 미리 만들어서 Content-Length를 계산하려고 했기 때문이지, 무조건 sprintf()가 필수라서가 아닙니다.
그리고 =로 바로 넣으면 안 된다고 한 이유는 content가 배열이기 때문입니다. 배열은 선언 후 문자열 전체를 =로 다시 대입할 수 없고, 함수로 내용을 써 넣어야 합니다.”

짧게 한 줄로 줄이면:

“sprintf()는 본문 길이 계산 때문에 쓴 것이고, =가 안 되는 건 content가 포인터가 아니라 배열이기 때문이다.”
