## 금요일 2026-04-17

### 목표

- `CSAPP` 11장 전체 구조를 본다.
- 최소한 "용어를 들어본 상태"가 아니라 "각 섹션이 왜 필요한지"를 말할 수 있어야 한다.
- 밤 24시 전 제출물도 끝낸다.

### 학습 범위

- 11.1 Client-Server Programming Model
- 11.2 Networks
- 11.3 The Global IP Internet
- 11.4 The Sockets Interface
- 11.5 Web Servers
- 11.6 Tiny Web Server
- 11.7 Summary

### 시간대 권장

- 오전:
  `11.1 ~ 11.3` 읽기 + 키워드 표시
- 오후:
  `11.4` 집중 학습
- 저녁:
  `11.5 ~ 11.6` 읽기 + Tiny 흐름 정리
- 밤:
  서로 오늘 어디까지 이해했는지 5~10분씩 공유

### 금요일 산출물

- `Client -> Server -> Response` 4단계 설명 1회
- `IP`, `DNS`, `socket`, `file descriptor`, `HTTP`를 자기 말로 2~3문장 정리
- Tiny가 static/dynamic을 어떻게 나누는지 한 장 요약
- 개인 GitHub Projects / Repo 제출 완료

## 어디까지 공부해야 충분한가

### 금요일 끝 기준

- 11장 전체 구조를 말할 수 있으면 통과
- 아직 구현은 못 해도 된다
- 대신 모르는 용어를 정확히 표시해야 한다

### 토요일 끝 기준

- 키워드 트리로 설명 가능해야 한다
- Tiny 흐름을 말로 풀 수 있어야 한다

### 월요일 끝 기준

- 질문과 답변이 실제로 생성되어 있어야 한다
- 구현 코드가 시작되어 있어야 한다

### 화요일 끝 기준

- API 서버 골격이 떠 있어야 한다
- 테스트 가능한 기능 최소 1개가 살아 있어야 한다

### 수요일 끝 기준

- 병렬 처리와 API 연결이 설명 가능한 수준까지 가 있어야 한다
- README만으로 데모가 가능한 상태여야 한다
