# 03. 학습 완료 판단표

이 문서는 "얼마나 공부했는가"가 아니라 "어디까지 설명, 구현, 검증할 수 있는가"를 판단하기 위한 기준입니다.

## 4단계 기준

- 1단계 `용어 인지`: 용어를 들어봤고 대략 뜻을 안다
- 2단계 `흐름 설명`: 그림이나 말로 전체 흐름을 설명할 수 있다
- 3단계 `코드 추적`: 함수와 시스템 호출 수준으로 따라갈 수 있다
- 4단계 `구현/수정`: 직접 만들거나 수정하고 검증할 수 있다

## 주제별 완료 기준

| 주제 | 2단계 기준 | 3단계 기준 | 4단계 기준 |
|------|-----------|-----------|-----------|
| Client-Server 모델 | 요청/응답 4단계를 설명 | Tiny/Proxy에 대응시켜 설명 | 서버 skeleton을 직접 짬 |
| IP / DNS / Port | host, IP, port 차이 설명 | socket address와 socket pair 설명 | 입력 URL에서 host/port/path를 파싱 |
| Sockets API | `socket -> bind -> listen -> accept` 설명 | 각 함수가 언제 호출되는지 코드로 추적 | 간단한 서버를 직접 띄움 |
| Client 연결 | `socket -> connect` 설명 | clientfd/open_clientfd 흐름 추적 | 테스트용 클라이언트 작성 |
| HTTP | request line, header, body 설명 | Tiny/Proxy 요청 처리 코드 추적 | 직접 요청 생성 및 디버깅 |
| Tiny 웹서버 | static/dynamic 차이 설명 | `parse_uri`, `serve_static`, `serve_dynamic` 추적 | Tiny 코드 일부 직접 수정 |
| Proxy | 브라우저-프록시-서버 흐름 설명 | header 재작성 규칙 추적 | sequential proxy 구현 |
| Concurrency | 왜 순차 서버가 느린지 설명 | thread-per-connection 설계 설명 | concurrent proxy 또는 thread pool 구현 |
| Cache | 캐시가 왜 필요한지 설명 | MAX_CACHE/MAX_OBJECT/LRU 요구사항 설명 | thread-safe cache 구현 |
| SQL API 서버 | DB 엔진과 API 서버 연결 구조 설명 | 요청 처리 경로를 함수 단위로 설명 | 병렬 SQL 요청 처리 구현 |

## 금요일 체크

아래 질문에 막히지 않으면 금요일 학습은 통과입니다.

- 클라이언트와 서버의 역할 차이를 1분 안에 설명할 수 있는가
- IP, DNS, port를 섞지 않고 설명할 수 있는가
- `socket`, `bind`, `listen`, `accept`의 순서를 이유와 함께 말할 수 있는가
- Tiny가 static과 dynamic을 어떻게 나누는지 설명할 수 있는가

## 토요일 체크

- 키워드 트리로 11장을 설명할 수 있는가
- "HTTP 요청이 Tiny에 들어와 응답이 나갈 때까지"를 말로 추적할 수 있는가
- 다른 사람 질문을 듣고 빠진 노드를 스스로 보완할 수 있는가

## 월요일 체크

- 이해가 안 되는 부분을 질문으로 바꿀 수 있는가
- 질문에 답할 때 코드/함수/흐름으로 근거를 댈 수 있는가
- 구현을 시작할 최소 서버 구조를 설명할 수 있는가

## 화요일 체크

- API 요청 1개를 실제로 받아 처리하고 응답하게 만들 수 있는가
- 동시성 문제가 어디서 생길지 미리 말할 수 있는가
- 테스트가 없는 기능을 불안하다고 느낄 정도로 감이 생겼는가

## 수요일 체크

- 외부 클라이언트가 붙을 수 있는 서버가 동작하는가
- 요청 처리 병렬화 전략을 설명할 수 있는가
- 생성된 코드의 핵심 로직을 직접 설명할 수 있는가
- README 기반으로 4분 데모가 가능한가

## 마지막 자기 점검 질문

- 내가 지금 설명하는 것이 "용어 정의"인가, "실행 흐름"인가, "구현 근거"인가
- 내가 고칠 수 없는 코드를 그냥 외우고 있지는 않은가
- 이 로직을 SQL API 서버에 붙일 때 어느 함수가 바뀌는지 말할 수 있는가
- 동시성, 캐시, 에러 처리를 아직 막연하게만 알고 있지는 않은가
